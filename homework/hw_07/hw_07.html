Good Plot

The good plot shows the same relationship more clearly

 good_madison <-filter(babynames, name == 'Madison')
glimpse(good_madison)
## Rows: 176
## Columns: 5
## $ year <dbl> 1880, 1881, 1882, 1883, 1884, 1885, 1886, 1887, 1888, 1889, 18...
## $ sex  <chr> "M", "M", "M", "M", "M", "M", "M", "M", "M", "M", "M", "M", "M...
## $ name <chr> "Madison", "Madison", "Madison", "Madison", "Madison", "Madiso...
## $ n    <int> 22, 28, 27, 22, 20, 20, 17, 18, 21, 23, 14, 20, 20, 12, 12, 15...
## $ prop <dbl> 0.00018581, 0.00025858, 0.00022126, 0.00019560, 0.00016295, 0....
goodplot <- ggplot(good_madison, aes(x = year, y = prop, color = sex)) + 
  geom_point() +
  xlab("Year") + 
  ylab("Proportion of total babies") +
  ggtitle("Babies named Madison over time") +
  scale_fill_discrete(name = "Gender", labels = c("Female", "Male"))+
  theme(plot.title = element_text(color="#666666", face="bold", size=15, hjust=0)) +
  theme(axis.title = element_text(color="#666666", face="bold", size=10))
  
goodplot

This plot also shows the distribution of babies named Madison over time, and the reltive proportion of babies named Madison that year, including both male and female.

This is a good plot because: 1) It shows a meaningful relationship.Here we can see the rarity of the name Madison as a boys name and that the name exploded as a girls name in the last 1900’s. Though the y-axis is in weird values, it provides us with the context of the relative popularity of the name. If we had for instance used the raw number of babies named Madison, we would have lost this clarity. 2) The text is legible and meaningful 3) The colors are descriptive 4) The graph is simple to show a simple relationship 5) The data visible

Unfortunately, this is not the best plot either. Some things that could be improved on: 1) This data should probably be further broken up into two figures. Due to the difference in magnitude between male and female. The number of males visually looks like it hovers around 0. Having a second zoomed in graph showing the male trend would be good. 2) For some reason the legend doesn’t format properly with scale_fill_discrete. 3) More axis lines could be added to improve data clarity, but I think this may clutter the graph. 4) The y-axis is in weird units. Further manipulation of the data prior to graphing could fix this. Maybe making the axis in units of 10^-3would streamline.
